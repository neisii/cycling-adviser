## 사이클링 날씨 어드바이저 — Production Rebuild

**Constraints confirmed:**
- Architecture: Vue 3 SPA + Cloudflare Workers proxy (already live)
- Scope: Korean market, 8 cities, Korean language

## 4.1 Product Requirements Document (PRD)

### Vision

서울 시민이 매일 아침 30초 안에 "오늘 자전거 탈 수 있나? 뭐 입고 나가야 하나?"에 답을 얻는 서비스.

> **One-line product statement:** 날씨 기반 자전거 탑승 가능성 판단 + 구체적인 라이딩 복장 추천 서비스.

---

### Personas

**P1 — 직장인 출퇴근 라이더 (Primary, 70%)**
- 매일 아침 7–8시 사용
- 세션 시간 < 30초
- 필요: 빠른 Go/No-Go + 오늘 뭐 입을지
- 날씨 데이터 자체에는 관심 없음

**P2 — 주말 레크리에이션 라이더 (Secondary, 25%)**
- 목~금 주말 계획 수립 시 사용
- 세션 시간 2–3분
- 필요: 3일 예보 + 날별 복장 계획
- 제품 속성(방수 여부 등)에 관심 있음

**P3 — 동호회 장거리 라이더 (Tertiary, 5%)**
- 주간 라이딩 전날 사용
- 세션 시간 3–5분
- 필요: 상세 날씨 조건 + 복장 레이어링 설명

---

### User Journeys

**Journey 1 — 출근 전 당일 체크 (P1)**
```
앱 열기
  → 저장된 도시 자동 표시 (기본: 서울)
  → 탑승 가능 여부 + 한 줄 이유 즉시 표시
  → 복장 카드 (아이템 3–5개) 표시
  → 완료
```

**Journey 2 — 주말 계획 수립 (P2)**
```
앱 열기
  → 도시 선택 (예: 부산)
  → 3일 탭 슬라이드
  → 각 날짜 탭 → 점수 + 복장 상세
  → 완료
```

**Journey 3 — 경기 전날 체크 (P3)**
```
앱 열기
  → 도시 선택
  → 상세 날씨 수치 확인 (기온/바람/강수/습도/자외선)
  → 복장 레이어별 상세 속성 확인 (보온지수, 방수 여부)
  → 완료
```

### Edge Cases

| 시나리오 | 기대 동작 |
|---|---|
| 3개 제공자 전부 실패 | KV 스테일 캐시 반환 (< 1시간); 없으면 503 + 한글 오류 메시지 |
| 제공자 부분 응답 (일부 필드 null) | 가용 필드로 점수 계산, `incomplete_data: true` 표시 |

| 강풍 > 15 m/s | 점수 −50 이상 페널티, 복장에 windproof 등급 강제 |
| Worker 응답 > 5초 | 프론트엔드 타임아웃 → 사용자 친화 오류 메시지 |

### Service Boundaries

| 레이어 | 책임 | 위치 |
|---|---|---|
| Vue SPA | UI 렌더링, 사용자 상호작용 | 브라우저 (Cloudflare Pages) |
| WeatherService | 제공자 호출, 캐시, 집계 | 프론트엔드 모듈 |
| WeatherAggregator | 가중 평균, 신뢰도 계산 | 프론트엔드 모듈 |
| CyclingScorer | 탑승 적합도 점수 계산 | 프론트엔드 모듈 |
| OutfitEngine | 복장 추천 로직 | 프론트엔드 모듈 |
| Worker Proxy | API 키 보호, 라우팅, 캐시 | Cloudflare Worker |

**핵심 원칙:** 비즈니스 로직(CyclingScorer, OutfitEngine)은 프론트엔드에 유지. Worker는 순수 프록시 역할만 담당.

---

### Data Flow

**실시간 요청 흐름:**
```
1. 사용자가 도시 선택 ("서울")
2. WeatherService.getCurrentWeather("seoul") 호출
3. localStorage 캐시 확인 → 히트 시 즉시 반환
4. 캐시 미스 → Worker Proxy에 3개 병렬 요청 (Promise.allSettled)
5. Worker가 각 외부 API 호출 (KV 캐시 확인 포함)
6. 응답 수집 (부분 실패 허용)
7. WeatherAggregator가 가중 평균 계산
8. CyclingScorer가 점수 산출
9. OutfitEngine이 복장 추천 생성
10. Pinia store 업데이트 → Vue 반응형 렌더링
```

**캐시 계층:**
```
Layer 1: localStorage (프론트엔드, TTL 5분)
  → 히트 시 Worker 호출 없음

Layer 2: Cloudflare KV (Worker, TTL 5분)
  → 히트 시 외부 API 호출 없음

Layer 3: 외부 API (실제 호출)
```

---

### Failure Handling

| 장애 유형 | 처리 전략 |
|---|---|
| 제공자 1개 타임아웃 (> 2초) | 나머지 2개로 계속 진행, `incomplete_data: true` |
| 제공자 2개 실패 | 남은 1개 데이터로 점수 계산, 경고 배너 표시 |
| 3개 전부 실패 | localStorage 스테일 캐시 (< 1시간) 반환; 없으면 오류 UI |
| Worker 5xx 응답 | 프론트엔드에서 재시도 1회, 실패 시 오류 메시지 |
| API 할당량 초과 | Worker에서 429 → 프론트엔드에서 다른 제공자로 폴백 |

## 4.4 API Design (Worker Proxy)

Worker는 순수 프록시이므로 "API 설계"는 Worker가 SPA에 노출하는 엔드포인트 계약.

### Base URL

`https://weather-proxy.neisii.workers.dev`

---

### `GET /openweather/current`

**Query params:** `lat`, `lon`, `lang=kr` (Worker가 주입)

**Worker 동작:**
1. KV에서 `ow:current:{lat}:{lon}` 확인
2. 히트 → 캐시 반환
3. 미스 → OpenWeatherMap API 호출 (API 키 Worker에서 주입)
4. 응답 KV 저장 (TTL: 300초)
5. SPA에 반환

**Response:** OpenWeatherMap 원본 응답 구조 (정규화는 프론트엔드에서 수행)

---

### `GET /openweather/forecast`

**Query params:** `lat`, `lon`, `cnt=24` (3일 데이터)

동일한 캐시·키 주입 패턴. TTL: 3600초.

---

### `GET /weatherapi/current`

**Query params:** `q={lat},{lon}`

---

### `GET /weatherapi/forecast`

**Query params:** `q={lat},{lon}`, `days=3`

---

### `GET /openmeteo/forecast`

**Query params:** `latitude`, `longitude`, `current=...`, `daily=...`, `timezone=Asia/Seoul`

Open-Meteo는 키 불필요. Worker는 CORS 헤더 및 캐시만 담당.

---

### Worker 오류 응답 (정규화)

```json

  "error": {

    "provider": "openweather",
    "status": 503,
    "message": "날씨 제공자에 연결할 수 없습니다."
```

## 4.5 Recommendation Engine Design

### CyclingScorer

```typescript
// src/services/cycling/CyclingScorer.ts

function score(weather: NormalizedWeather): CyclingResult {
  // 1. 극단 조건 단락 평가
  if (isExtreme(weather)) return EXTREME_RESULT

  // 2. 요소별 점수 계산
  const breakdown = {
    temperature:   scoreTemperature(weather.temp),      // max deduct: -40
    wind:          scoreWind(weather.wind_speed),        // max deduct: -30
    precipitation: scorePrecipitation(weather),          // max deduct: -50
    humidity:      scoreHumidity(weather.humidity),      // max deduct: -15
    uv:            scoreUV(weather.uv_index),            // max deduct: -10
  }

  // 3. 합산 (base 100, 최소 0)
  const raw = 100 + Object.values(breakdown).reduce((a, b) => a + b, 0)
  const score = Math.max(0, Math.min(100, raw))

  return {
    score,
    verdict: toVerdict(score),
    verdict_ko: VERDICT_KO[toVerdict(score)],
    score_breakdown: breakdown,
    reasons: buildReasons(breakdown, weather),
    reasons_ko: buildReasonsKo(breakdown, weather),
    outfit: OutfitEngine.recommend(weather),
  }
}
```

**점수 공식 상세:**

| 요소 | 이상 범위 | 감점 공식 |
|---|---|---|
| 기온 | 10–24°C | °C 초과당 -5, 상한 -40 |
| 풍속 | < 6 m/s | m/s 초과당 -5 (6 기준), 상한 -30 |
| 강수 | 0 mm | drizzle: -15, light_rain: -25, moderate: -40, heavy/storm: -50 |
| 습도 | 30–80% | 10% 초과당 -3 (범위 밖 기준), 상한 -15 |
| 자외선 | < 8 | 8 초과 시 -5, 10 초과 시 추가 -5 |
| 극단 | condition = extreme | score 강제 0 |

**판정 매핑:**
```
90–100 → excellent / 최상
70–89  → good      / 좋음
50–69  → fair      / 보통
30–49  → poor      / 나쁨
0–29   → dangerous / 위험
```

### OutfitEngine

```typescript
// src/services/cycling/OutfitEngine.ts

function recommend(weather: NormalizedWeather): OutfitRecommendation {
  const items: OutfitItem[] = []

  // 1. 헬멧 (항상 필수)
  items.push(catalog.get('helmet'))

  // 2. 상의 레이어 (기온 기반)
  items.push(...selectTorsoLayers(weather.temp, weather.wind_speed))

  // 3. 하의 (기온 기반)
  items.push(...selectLegsLayers(weather.temp))

  // 4. 발 (기온 + 강수 기반)
  items.push(...selectFeet(weather.temp, weather.condition))

  // 5. 조건부 추가
  if (isRainy(weather.condition))   items.push(catalog.get('waterproof_jacket'))
  if (weather.wind_speed > 8)       items.push(catalog.get('windproof_jacket'))
  if ((weather.uv_index ?? 0) > 3)  items.push(catalog.get('sunscreen'))
  if ((weather.uv_index ?? 0) > 6)  items.push(catalog.get('sun_cap'))
  if (weather.condition === 'fog')  items.push(catalog.get('front_rear_light'))

  return groupByCategory(items)
}
```

**상의 선택 규칙:**

| 기온 | 필수 레이어 | 선택 레이어 |
|---|---|---|
| > 24°C | 반팔 저지 | 선크림, 선글라스 |
| 16–24°C | 반팔 저지 | 암워머 (20°C 미만 시) |
| 10–15°C | 반팔 저지 + 암워머 | 경량 윈드베스트 |
| 5–9°C | 긴소매 저지 | 윈드재킷 또는 기모 조끼 |
| 0–4°C | 베이스레이어 + 기모 저지 | 방풍 재킷 |
| < 0°C | 베이스레이어 + 기모 저지 + 방풍 재킷 | — |

**하의 선택 규칙:**

| 기온 | 아이템 |
|---|---|
| > 16°C | 빕숏츠 |
| 10–15°C | 빕숏츠 + 니워머 |
| < 10°C | 기모 빕타이츠 |

**발 선택 규칙:**

| 조건 | 아이템 |
|---|---|
| 기온 > 15°C, 맑음 | 일반 사이클링 양말 |
| 기온 10–15°C | 두꺼운 사이클링 양말 |
| 기온 < 10°C | 오버슈즈 |
| 강수 있음 | 방수 오버슈즈 |

- 기존 `CustomWeatherPredictor.ts` 집계 로직 재사용
